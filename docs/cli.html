cancan <group> <command>——按命令组列出全部子命令与参数。等价写法：python -m cancan_microstack.cli。 cancan <group> <command> — every subcommand and flag, grouped. Equivalent form: python -m cancan_microstack.cli.

命令组Command groups
init doctor assets compose services stack controllersrv version
为什么 stack 同时管两层？controllersrv 运行在宿主机——它要直接操作 Docker/Podman，无法由 compose 管理；其余服务都是容器。因此 stack up/down/status 会同时管理宿主机进程（pidfile 守护）与容器栈（compose）。 Why does stack manage two layers? controllersrv runs on the host — it drives Docker/Podman directly and cannot be managed by compose; everything else is a container. So stack up/down/status manages both the host process (a pidfile-tracked daemon) and the container stack (compose).

init — 脚手架最小可跑工作区init — scaffold a minimal runnable workspace

装完即用：生成业务 compose 样例、.env、overrides 骨架等。From install to runnable: a services compose sample, .env, an overrides skeleton, and more.

cancan init [--workspace .]

在目标工作区生成：In the target workspace it creates:

  • docker-compose.services.yml — 业务服务样例a business-services sample
  • .env — 配置与密钥（自动写入唯一的 TOTP 加密 key）config & secrets (a unique TOTP encryption key is generated for you)
  • cancan_overrides/ — 免 fork 覆盖骨架a fork-free override skeleton
  • 以及 infra compose / Caddy·service Dockerfile / DDL / adminops 等运行时资产plus runtime assets: infra compose, Caddy/service Dockerfiles, DDL, adminops

doctor — 起栈前环境与配置预检doctor — pre-flight environment & config checks

一次性诊断常见的"起不来"原因。Diagnose the common "won't come up" causes in one shot.

cancan doctor [--workspace .]

检查项：It checks:

  • 容器引擎与 daemon 是否可用（docker / podman）container engine & daemon availability (docker / podman)
  • 宿主端口是否被占用host port conflicts
  • 工作区是否可识别、运行时资产是否就绪workspace detection & bootstrapped runtime assets
  • AUTH_TOTP_FERNET_KEY 是否设置；是否仍在用弱默认口令whether AUTH_TOTP_FERNET_KEY is set and whether weak default secrets remain

有阻塞问题返回非零退出码，便于做 CI 门禁。Returns a non-zero exit code on blocking issues — handy as a CI gate.

assets — 检查与导出包内资产assets — inspect and export packaged assets

列出随库发布的静态文件，或把某个资产导出到工作区。List the static files shipped with the library, or export one into the workspace.

命令Command作用Purpose关键参数Key arguments
assets list [subdir] 列出资产；可选位置参数 subdir 限定子目录。List assets; the optional positional subdir filters to a subdirectory. subdir （可选，位置参数）(optional, positional)
assets export <logical_name> <destination> 把资产复制到目标路径；目录用 copytree、文件用 copy2。Copy an asset to a destination; directories via copytree, files via copy2. logical_name, destination （位置参数）(positional); --overwrite 
# 列出全部资产 / List all assets
cancan assets list

# 仅 docker 子目录 / Only the docker subdir
cancan assets list docker

# 导出 infra compose 到工作区，覆盖已存在文件
# Export the infra compose into the workspace, overwriting if present
cancan assets export docker/docker-compose.infra.yml ./infra/docker-compose.infra.yml --overwrite

--overwrite 未指定且目标已存在时，导出会报 FileExistsError。Without --overwrite, exporting onto an existing destination raises FileExistsError.

compose — 合成 compose 栈文件compose — synthesize the compose stack file

以内置 infra compose 为基底，叠加业务 compose 与 override，输出一个可运行栈。Take the built-in infra compose as the base, layer in your services compose and overrides, and emit one runnable stack.

命令Command作用Purpose
compose build生成合并后的 compose 文件。Generate the merged compose file.

参数Arguments

参数Flag默认Default作用Purpose
--workspace.工作区根目录。Workspace root.
--service-file—业务 compose 文件（可选）。Business compose file (optional).
--override[]额外 override compose 文件，可重复多次。Extra override compose files; repeatable.
--output<workspace>/compose.cancan.yml目标 compose 文件路径。Target compose file path.
--bootstrap / --no-bootstrap--bootstrap是否生成缺失的工作区运行文件（cmd/* 包装器、infra compose 导出等）。Whether to generate missing workspace run files (cmd/* wrappers, infra compose export, …).

合成规则：infra compose（基底）→ 业务 compose → override，后者深度合并并覆盖前者的同名字段。默认输出 <workspace>/compose.cancan.yml（除非显式 --output）。 Merge order: infra compose (base) → services compose → overrides, each deep-merged and overriding same-named fields of the previous. The default output is <workspace>/compose.cancan.yml unless you pass --output. 

# 业务 compose + 两份 override，自定义输出路径
# Services compose + two overrides, custom output path
cancan compose build \
  --workspace . \
  --service-file docker-compose.services.yml \
  --override compose.local.yml \
  --override compose.debug.yml \
  --output compose.cancan.yml

services — 前台运行内置服务services — run a bundled service in the foreground

直接前台启动单个服务，适合开发调试；启动前自动配置 workspace 与 overrides。Run a single service in the foreground — ideal for dev and debugging; the workspace and overrides are configured automatically before startup.

命令Command作用Purpose
services run <name>前台运行一个内置服务。name ∈ {controllersrv, infrasrv, opsbffsrv}。Run one bundled service in the foreground. name ∈ {controllersrv, infrasrv, opsbffsrv}.

参数Arguments

参数Flag默认Default作用Purpose
name （位置参数）(positional)—服务名，限三选一。Service name; one of the three.
--host0.0.0.0绑定主机。Bind host.
--port按服务而定per-service绑定端口（controllersrv 默认 22100，infrasrv / opsbffsrv 默认 8080）。Bind port (controllersrv defaults to 22100; infrasrv / opsbffsrv to 8080).
--workspace.工作区根目录。Workspace root.
 
# 前台跑 infrasrv 调试 / Run infrasrv in the foreground for debugging
cancan services run infrasrv --workspace . --port 8080

同时管理宿主机 controllersrv 与容器栈：up / down / status。Manage the host controllersrv and the container stack together: up / down / status.

stack up — 合成 + bootstrap + 起 controllersrv + compose upbuild + bootstrap + start controllersrv + compose up

--override[]额外 override 文件，可重复。Extra override files; repeatable.

--bootstrap / --no-bootstrap--bootstrap是否生成缺失的工作区运行文件（含 adminops 资产）。Generate missing workspace run files (including adminops assets).
--with-controllersrv / --no-with-controllersrv--with-controllersrv起容器前是否确保宿主机 controllersrv 在运行。Ensure the host controllersrv is running before bringing up containers.

controllersrv — 管理宿主机守护进程controllersrv — manage the host daemon

单独重启在宿主机后台运行的 controllersrv 守护进程。Restart the controllersrv daemon that runs in the background on the host.

命令Command作用Purpose
controllersrv restart先停止已有守护进程（若有），再在宿主机启动一个全新的 controllersrv。Stop the existing daemon (if any), then start a fresh controllersrv on the host.

参数Arguments

参数Flag默认Default作用Purpose
