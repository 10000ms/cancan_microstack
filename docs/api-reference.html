统一响应约定Response envelope

读懂一次，全站通用——下面所有接口都遵循同一个包装。Learn it once: every endpoint below follows the same wrapper.

统一 APIResponse 包装。所有响应都包成 { success, error: { code, msg }, data }：success 表示业务是否成功，data 是业务载荷，错误经由统一 ErrorCode 表达（0=成功、4000=参数错误、4001=未登录、4004=未找到、5000=系统错误、5001=服务管理错误、5100=网络错误等）。下表只列方法、路径与用途，不重复展开包装。 Unified APIResponse wrapper. Every response is wrapped as { success, error: { code, msg }, data }: success is the business outcome, data is the payload, and failures are expressed through a unified ErrorCode (0 = success, 4000 = invalid param, 4001 = not logged in, 4004 = not found, 5000 = system error, 5001 = service-management error, 5100 = network error, …). The tables below list method, path and purpose only — they don't repeat the wrapper.

路径中的 {…} 为路径参数（如 {workflow_id}、{id:uuid}、{path:path} 通配子路径）。所有接口都挂在 Caddy 网关之后，外部统一经网关访问，不直接暴露服务端口。 A {…} in a path is a path parameter (e.g. {workflow_id}, {id:uuid}, or {path:path} for a wildcard sub-path). All endpoints sit behind the Caddy gateway; external traffic goes through the gateway rather than hitting service ports directly.
