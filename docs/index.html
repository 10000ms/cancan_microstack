Cancan Microstack

把一整套基础设施微服务（controllersrv / infrasrv / opsbffsrv）、一个可视化工作流引擎、一套运维管理前端，连同 Docker Compose 编排、数据库 DDL 与统一 CLI，打包成一个可 pip 安装的 Python 库——让业务仓库“只写业务代码”。 Bundle a complete set of infrastructure microservices (controllersrv / infrasrv / opsbffsrv), a visual workflow engine, an ops admin UI — together with Docker Compose orchestration, database DDL and a unified CLI — into a single pip-installable Python library, so product repos contain business logic only.

这是什么What it is

一个“资产 + 合成 + 运行”三合一的微服务工具箱，并自带一整套运维控制面。A microstack toolkit unifying assets, compose synthesis and service running — with a full ops control plane built in.

在每个业务仓里反复搭基础设施（拷 compose、维护 DDL、起支撑服务）既重复又易漂移。Cancan 把这些固化成一个可升级、可分发的库：基础设施随库版本演进，而不是到处 copy-paste。它不仅给你现成的服务，还给你管理它们的工作流引擎与运维前端。 Rebuilding infrastructure in every product repo (copying compose, maintaining DDL, standing up supporting services) is repetitive and drifts. Cancan freezes all of it into an upgradeable, distributable library: infrastructure evolves with a library version instead of copy-paste. Beyond ready-made services, it gives you a workflow engine and an ops UI to operate them.
