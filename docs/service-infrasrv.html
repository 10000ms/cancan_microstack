infrasrv — 基础设施服务Infrastructure Service

栈内的基础设施中枢：聚合服务配置管理、服务注册与发现、健康检查、服务管理，以及操作（operation）与实例（instance）编排和工作流引擎。controllersrv 与 opsbffsrv 都把状态托管给它。 The infrastructure hub inside the stack: it aggregates service config, registry & discovery, health checks, service management, plus operation and instance orchestration and the workflow engine. Both controllersrv and opsbffsrv park their state here.

概述Overview

微服务栈的协调面：谁在哪、是否健康、配置是什么、操作进行到哪一步。The stack's coordination plane: who is where, is it healthy, what is its config, and where an operation stands.

infrasrv 把分散的“运行时事实”集中持久化：服务的配置（service_config）、注册实例（service_instance / service_info）、异步操作记录（service_operation）、动作日志（service_action_log），以及工作流定义与运行。controllersrv 执行容器操作时，回写操作/实例状态到这里；opsbffsrv 则从这里读取概览供前端展示。 infrasrv centralises and persists the scattered "runtime facts": service config (service_config), registered instances (service_instance / service_info), async operation records (service_operation), action logs (service_action_log), plus workflow definitions and runs. When controllersrv runs a container operation it writes operation/instance state back here; opsbffsrv reads the overview from here for the UI.

实例注册 / 注销 / 查询，缓存加持，配合批量健康检查。Instance register / deregister / lookup, cache-backed, with batch health checks.

内部编排面Internal control plane

internal/operation/* 与 internal/instance/* 供 controllersrv 回写状态、分配端口、统计副本。internal/operation/* and internal/instance/* let controllersrv write back state, allocate ports and track replicas.

工作流引擎Workflow engine

工作流定义、版本、运行与告警的领域层就在 infrasrv，详见工作流系列页面。The domain layer for workflow definitions, versions, runs and alerts lives in infrasrv — see the workflow pages.

命名约定：带 internal/ 前缀的接口是服务间内部接口（主要供 controllersrv 调用），不直接面向前端；其余接口供 opsbffsrv 聚合或服务自身使用。所有响应为 APIResponse 信封，下表“关键字段”指 data 内容。工作流引擎 API 不在本页展开。 Convention: endpoints prefixed internal/ are service-to-service (chiefly called by controllersrv), not front-facing; the rest serve opsbffsrv aggregation or the service itself. Responses use the APIResponse envelope; "key fields" below describe data. Workflow-engine endpoints are not expanded here.
工作流引擎 API：infrasrv 还承载 /v1/infrasrv/workflows*、/runs* 与 /callbacks/{node_instance_id} 等工作流接口（定义、版本、回滚、触发、运行图状态、节点历史、外部回调、引擎告警）。这些不在本页详述，请见 工作流·概念与状态机 与 工作流·API 参考。 Workflow engine API: infrasrv also hosts the workflow endpoints /v1/infrasrv/workflows*, /runs* and /callbacks/{node_instance_id} (definitions, versions, rollback, trigger, run-graph status, node history, external callback, engine alerts). These are covered not here but in Workflow Concepts & State and Workflow API.

服务注册与发现 APIRegistry & Discovery API

实例注册、注销、查询，加批量健康检查。Register, deregister and query instances, plus batch health checks.

方法Method 路径Path 用途Purpose 关键字段Key fields
POST/v1/infrasrv/registry/register 注册实例Register an instance 请求Req: service_name, instance_id, host, port, internal_port, service_type, health_check_path, desired_replicas, container_name, compose_service_name, config_version, service_metadata …
DELETE/v1/infrasrv/registry/deregister 注销实例Deregister an instance 请求Req: service_name, instance_id (query)
GET/v1/infrasrv/registry/instances 查某服务的实例List a service's instances 请求Req: service_name, only_healthy (def true) · 响应Resp: instances[]
POST/v1/infrasrv/health/check_instances 批量健康检查所有实例Batch health-check all instances 响应Resp: 检查结果汇总check-result summary

register 的请求体很宽：除地址（host/port/internal_port）外，还携带服务画像（service_type、owner、deploy_region）、副本意图（desired_replicas、scale_policy）、健康配置（health_check_path、health_check_url）与容器关联（container_name、compose_service_name）。注册信息被缓存以加速发现查询。 register takes a broad body: beyond the address (host/port/internal_port) it carries the service profile (service_type, owner, deploy_region), replica intent (desired_replicas, scale_policy), health config (health_check_path, health_check_url) and container linkage (container_name, compose_service_name). Registrations are cached to speed up discovery lookups.

operation/status 与 operation/get 共用同一 handler，是同一查询的两个别名路径。 operation/status and operation/get share the same handler — two alias paths for one query.
